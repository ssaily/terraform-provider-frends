## Provider Configuration

**Example** · [`examples/provider/provider.tf`](../examples/provider/provider.tf)

```hcl
terraform {
  required_providers {
    frends = {
      source  = "frends/frends"
      version = "~> 0.1"
    }
  }
}

provider "frends" {
  host_url = "https://my-company.frends.com"
  token    = var.frends_token
}
```

### Arguments

| Argument | Required | Description |
|---|---|---|
| `host_url` | Yes | Base URL of your Frends Platform instance. Also set via `FRENDS_HOST_URL` env var. |
| `token` | Yes | Bearer token for API authentication. Also set via `FRENDS_TOKEN` env var. |

## Resources

### frends_api_key

Manages a Frends API access key for a specific environment. The key value (UUID) is generated by the API and marked sensitive.

**Example** · [`examples/resources/api_key/resource.tf`](../examples/resources/api_key/resource.tf)

#### Operations

| Operation | Behaviour |
|---|---|
| **Create** | Generates a new API key in the given environment. The `value` UUID is only available after create. |
| **Read** | Refreshes `name`, `value`, `modified`, `modifier`, and `environment_id` from the API. |
| **Update** | Updates the key `name`. The `value` and `environment_id` are immutable after creation. |
| **Delete** | Deletes the API key from the environment. |
| **Import** | `terraform import frends_api_key.example <id>` — numeric key ID. |

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | Optional | Human-readable label for the key. |
| `environment_id` | number | **Required** | ID of the environment. **Changing this forces replacement.** |

#### Attributes (Computed)

| Attribute | Type | Sensitive | Description |
|---|---|---|---|
| `id` | number | — | Unique identifier of the API key. |
| `value` | string | **Yes** | The API key UUID. Only available after create. |
| `modified` | string | — | Timestamp of last modification. |
| `modifier` | string | — | User who last modified the key. |

### frends_api_policy

Manages a Frends API Policy that controls access to API endpoints via API keys or JWT identity claim rules.

**Example** · [`examples/resources/api_policy/resource.tf`](../examples/resources/api_policy/resource.tf)

#### Operations

| Operation | Behaviour |
|---|---|
| **Create** | Creates an API policy with the supplied endpoint targets, identities, and tags. |
| **Read** | Refreshes `name`, `description`, `allow_public_access`, `api_key_name`, and `api_key_location` from the API. Tags, target endpoints, and identities are not re-read (plan-level blocks). |
| **Update** | Replaces the full policy definition (name, endpoints, identities, tags). |
| **Delete** | Deletes the policy. |
| **Import** | `terraform import frends_api_policy.example <id>` — numeric policy ID. Tags, endpoints, and identities will be empty after import and must be configured manually. |

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | **Required** | Policy name. |
| `description` | string | Optional | Human-readable description. |
| `allow_public_access` | bool | Optional | Allow unauthenticated access. Default: `false`. |
| `api_key_name` | string | Optional | Name of the API key header/query parameter. |
| `api_key_location` | string | Optional | Where the API key is expected: `Header` or `Query`. |
| `tags` | list(string) | Optional | Arbitrary tags. |

#### Block: `target_endpoints`

Repeatable. Defines which API endpoints this policy protects.

| Argument | Type | Required | Description |
|---|---|---|---|
| `url` | string | **Required** | Full endpoint URL. |
| `method` | string | Optional | HTTP method (e.g. `GET`, `POST`). Omit for all methods. |

#### Block: `identities`

Repeatable. Defines identities (principals) allowed to call the protected endpoints.

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | **Required** | Identity label. |

##### Block: `identities.rules`

Repeatable. JWT claim-based access rules for the identity.

| Argument | Type | Required | Description |
|---|---|---|---|
| `claim_type` | string | **Required** | JWT claim key (e.g. `aud`, `roles`). |
| `claim_value` | string | **Required** | Expected claim value. |
| `match_type` | string | Optional | Comparison strategy (e.g. `Exact`, `Contains`). |

#### Attributes (Computed)

| Attribute | Type | Description |
|---|---|---|
| `id` | number | Unique identifier of the policy. |

### frends_environment_variable

Manages a Frends environment variable schema and its per-environment values. The schema (name, type, description) is shared across all environments; values are set per-environment via `values` blocks.

**Example** · [`examples/resources/environment_variable/resource.tf`](../examples/resources/environment_variable/resource.tf)

#### Operations

| Operation | Behaviour |
|---|---|
| **Create** | Creates the variable schema, then sets each per-environment value. |
| **Read** | Refreshes `name`, `type`, and all per-environment values from the API. |
| **Update** | Re-applies all `values` blocks. Values not listed are not changed (they are additive from the API's perspective). |
| **Delete** | Deletes the variable schema and all its values. |
| **Import** | `terraform import frends_environment_variable.example <id>` — numeric schema ID. The `values` block will be empty after import. |

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | **Required** | Variable name. **Changing this forces replacement.** |
| `type` | string | Optional/Computed | Variable type. One of: `String`, `Object`, `Array`, `Secret`, `Number`, `Boolean`. Defaults to `String` if not set. |
| `description` | string | Optional | Human-readable description. |

#### Block: `values`

Repeatable. Sets the variable's value for a specific environment.

| Argument | Type | Required | Description |
|---|---|---|---|
| `environment_id` | number | **Required** | ID of the target environment. |
| `value` | string | **Required** | JSON-encoded value for this environment. |

#### Attributes (Computed)

| Attribute | Type | Description |
|---|---|---|
| `id` | number | Unique identifier of the variable schema. |

### frends_private_application

Manages a Frends Private Application used for OAuth token-based access. Private applications define token lifetime and custom JWT claims.

**Example** · [`examples/resources/private_application/resource.tf`](../examples/resources/private_application/resource.tf)

#### Operations

| Operation | Behaviour |
|---|---|
| **Create** | Creates the private application with claims, tags, and token lifetime. |
| **Read** | Refreshes `name`, `description`, `default_token_lifetime_days`, `modifier`, and `modified_utc`. Custom claims and tags are not re-read. |
| **Update** | Replaces name, description, token lifetime, claims, and tags. |
| **Delete** | Deletes the application. |
| **Import** | `terraform import frends_private_application.example <id>` — numeric application ID. `custom_token_claims` and `tags` will be empty after import. |

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | **Required** | Application name. |
| `description` | string | Optional | Human-readable description. |
| `default_token_lifetime_days` | number | **Required** | Token lifetime in days (1–730). |
| `custom_token_claims` | map(string) | Optional/Computed | Key/value map of custom JWT claims to embed in generated tokens. |
| `tags` | list(string) | Optional | Arbitrary tags. |

#### Attributes (Computed)

| Attribute | Type | Description |
|---|---|---|
| `id` | number | Unique identifier of the application. |
| `modifier` | string | User who last modified the application. |
| `modified_utc` | string | ISO 8601 timestamp of last modification. |

### frends_process_deployment

Manages a Frends process deployment — deploys one or more process versions to an agent group, optionally with per-process variable overrides.

> **Force-replace note:** Changing `agent_group_id` or any `processes` block forces Terraform to destroy and recreate the deployment.

**Example** · [`examples/resources/process_deployment/resource.tf`](../examples/resources/process_deployment/resource.tf)

#### Operations

| Operation | Behaviour |
|---|---|
| **Create** | Deploys the listed process versions to the specified agent group. Triggers are activated according to `activate_triggers`. |
| **Read** | Refreshes `triggers_active` from the API. Process list and agent group are not re-read (sourced from state). |
| **Update** | If `activate_triggers` changed, updates the trigger activation state. Process list and agent group changes force replacement instead. |
| **Delete** | Removes the deployment from the agent group. |
| **Import** | `terraform import frends_process_deployment.example <id>` — numeric deployment ID. The `processes` block will be empty after import. |

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `agent_group_id` | number | **Required** | ID of the target Agent Group. **Changing this forces replacement.** |
| `activate_triggers` | bool | Optional/Computed | Activate triggers on deployment. Default: `true`. |
| `deployment_description` | string | Optional | Free-text description for this deployment. |

#### Block: `processes`

Repeatable. Specifies which process versions to deploy. **Changing this block forces replacement.**

| Argument | Type | Required | Description |
|---|---|---|---|
| `process_guid` | string | **Required** | GUID of the process (shared across versions). **Changing this forces replacement.** |
| `version` | number | **Required** | Build version number to deploy. |

##### Block: `processes.process_variables`

Repeatable. Override process-level variables for this deployment.

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | **Required** | Variable name. |
| `value` | string | Optional | Variable value. |
| `is_secret` | bool | Optional/Computed | Mark as secret (value hidden in Frends UI). |
| `description` | string | Optional | Variable description. |

#### Attributes (Computed)

| Attribute | Type | Description |
|---|---|---|
| `id` | number | Unique identifier of the deployment. |
| `triggers_active` | bool | Current trigger activation state as reported by the API. |

Fetches a Frends Agent Group by display name within a specific environment. Use this to obtain the `id` needed for `frends_process_deployment`.

**Example** · See [`examples/resources/process_deployment/resource.tf`](../examples/resources/process_deployment/resource.tf)

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `display_name` | string | **Required** | Display name of the agent group to look up. |
| `environment_id` | number | **Required** | ID of the environment that contains this agent group. |

#### Attributes (Computed)

| Attribute | Type | Description |
|---|---|---|
| `id` | number | Unique identifier of the agent group. |
| `internal_name` | string | Internal name used by Frends. |
| `is_cross_platform` | bool | Whether the group supports cross-platform agents. |
| `agents` | list(string) | Names of agents registered in this group. |

### frends_process

Fetches a Frends Process by name. Processes are built in Frends Control Panel and are read-only in Terraform. Use this to obtain the `guid` and `version` needed for `frends_process_deployment`.

Only non-draft, non-deleted processes are returned.

**Example** · See [`examples/resources/process_deployment/resource.tf`](../examples/resources/process_deployment/resource.tf)

#### Arguments

| Argument | Type | Required | Description |
|---|---|---|---|
| `name` | string | **Required** | Exact name of the process. |

#### Attributes (Computed)

| Attribute | Type | Description |
|---|---|---|
| `id` | number | Unique identifier of the process version record. |
| `guid` | string | GUID shared across all versions of this process. Pass this to `processes.process_guid`. |
| `version` | number | Current build version. Pass this to `processes.version`. |
| `description` | string | Process description. |
| `is_draft` | bool | Whether the process is a draft (always `false` for returned results). |
